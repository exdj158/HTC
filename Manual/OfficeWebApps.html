Integration with Microsoft Office Web Apps Server

Microsoft Office Web Apps Server can be used for viewing Office documents (Word, Excel, PowerPoint) in browser from HTTPCommander without Microsoft Office installed on the computer.

The following options are possible:
  • To use Microsoft Office Web Apps online viewer without Office Web Apps Server installation in an intranet
    For this purpose it is required that site with HTTPCommander was available from out of on Internet (hidden parameter DomainName can be here too used); anonymous access to AnonymousDownload.ashx is configured (see How to grant the anonymous access to AnonymousDownload.ashx), and computers of users had Internet access.
    Further it is necessary:
    • to set EnableOWADocumentsViewer parameter value in true;
    • for OWADocumentsViewerUrl and OWADocumentsViewerEmbeddedUrl set defaults values:
      "https://view.officeapps.live.com/op/view.aspx?src="
      and
      "https://view.officeapps.live.com/op/embed.aspx?src= "
      accordingly.
    For more info see View Office documents online.
  • To install and configure Office Web Apps Server in an intranet
    In this case access to Internet from the user computers isn't required. Also it isn't required access from out of on Internet to a site with HTTPCommander. But setup of anonymous access is required to AnonymousDownload.ashx page (see How to grant the anonymous access to AnonymousDownload.ashx).
    Office Web Apps Server is a new Office server product that delivers browser-based versions of Word, PowerPoint, Excel, and OneNote. Following the instruction, install and configure Office Web Apps Server on one of Your suitable servers (it is necessary that server was joined to domain) on local area network:
    • Install prerequisite software for Office Web Apps Server.
      It is also necessary that on server Microsoft Office wasn't installed.
    • Download and install Office Web Apps Server.
      Note. To download Office Web Apps Server you must have a license, under a Volume Licensing agreement, for Office Professional Plus 2013, Office Standard 2013, or Office for Mac 2011. The download is located under those Office products on the VLSC portal.
      Download and install language packs for Office Web Apps Server (if multilanguage support is required).
    • Follow the procedures in one of the following three sections, based on what kind of Office Web Apps Server farm you want to create.
      In the elementary option to create a new Office Web Apps Server farm that consists of a single server, as shown in the following example (command needs to be entered into Windows PowerShell):
      New-OfficeWebAppsFarm -InternalURL "http://YourServerNameWithInstalledOfficeWebApps" -AllowHttp -OpenFromUncEnabled:$true -OpenFromUrlEnabled:$true
    • For check of the correct installation, open in the browser the page
      http://YourServerNameWithInstalledOfficeWebApps/op/generate.aspx
      See screenshot

       


      If the page didn't open, address to documentation.
    • Also check, whether site with HTTP Commander from server where Office Web Apps Server was installed opens.
    Further it is necessary:
    • to set EnableOWADocumentsViewer parameter value in true;
    • for OWADocumentsViewerUrl and OWADocumentsViewerEmbeddedUrl set values:
      "http://YourServerNameWithInstalledOfficeWebApps/op/view.aspx?src="
      and
      "http://YourServerNameWithInstalledOfficeWebApps/op/embed.aspx?src="
      accordingly.
    For more info see Deploy Office Web Apps Server.

After that in the 'View and edit' menu item of file context menu in the HTTP Commander, will be available menu item "View in Office Web Apps" for documents listed in OWADocumentsViewerFileTypes. The list of supported extensions of files you can find (and change) in OWADocumentsViewerFileTypes parameter.
