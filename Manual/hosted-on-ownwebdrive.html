HTTP Commander hosted on ownwebdrive.com.

In this section we will provide some details about Http Commander hosted on ownwebdrive.com.
If you decided to use "hosted by us" version of Http Commander, you will receive access to already installed and pre-configured version of Http Commander on a virtual server.

How to access it?

You can access the application by navigating to address https://yourcompany.ownwebdrive where "yourcompany" is a company/team name used during the registration.
Use admin/admin username/password to log into the application with administrator privileges.
Do not forget to change the default password once you logged in. Also change your password for default user account "demo" or remove it.

Where is it located?

  • C:\Inetpub\wwwroot\HTCOMNET - Http Commander application installed to this folder. Root folder for "Default Web Site" is point to this folder, is a point to this folder, therefore when you open https://yourcompany.ownwebdrive in browser (without /htcomnet), you will see the application loaded.
  • C:\HttpCommanderData - HTTP Commander stores the configuration data (user accounts, folders config, error log, public links) in this folder. For security reasons, it is recommended to prevent users from retrieving them, since they expose sensitive information about the application. Therefore we moved Http Commander "Data" folder outside of application folder to "C:\HttpCommanderData".
  • C:\HttpCommanderSharedFolder - folder with default shared folders in Http Commander.
    Of course you can share in Http Commander any folder, not only folders from "C:\HttpCommanderSharedFolder". But it is not recommended to share folders from application folder C:\Inetpub\wwwroot\HTCOMNET.
    • On both of C:\HttpCommanderSharedFolder and C:\HttpCommanderSharedFolder folders read/write access granted for IIS APPPOOL\htcomnetpool user.

What is preconfigured?

  • Root folder for "Default Web Site" is a point to C:\Inetpub\wwwroot\HTCOMNET folder.
  • Application pool htcomnetpool assigned to "Default Web Site".
  • SSL certificate installed and SSL binding on 443 port applied for "Default Web Site".
  • We configured the integration with following cloud storages and online services:
    • Google Drive - used for "Edit in Google Drive", Upload/Download from/to Google Drive features
    • MS One drive - used for "Edit in Ms Office Online", Upload/Download from/to MS OneDrive features
    • Dropbox - used for Upload/Download from/to Dropbox feature
    • Box - used for "View in Box", Upload/Download from/to Box features
    • Zoho - used for "Edit in Zoho" feature
    We used our API keys for configuration. If you would like to use your own accounts (API keys), feel free to re-configure Http Commander. You can follow the instructions from documentation.
  • • WebDav access to shared folders is configured. You can disable it by setting to false the value of the EnableWebFoldersLinks setting and remove FileWebDavModule module from web.config.
  • Configured permissions for IIS APPPOOL\htcomnetpool user on following folders:
    • C:\Inetpub\wwwroot\HTCOMNET - read
    • C:\HttpCommanderData - read/write
    • C:\HttpCommanderSharedFolder - read/write
  • Url-Rewrite IIS module installed and HTTP-to-HTTPS redirect rule configured for "Default Web Site".
  • SMTP server installed and Http Commander configured to use it to send notification emails. It is done only to make it work out of the box. Please configure Http Commander mail settings to use your corporate smtp server.
