Interface view modes

HTTP Commander can work in a stand-alone window and inside iframe. You can also use iframe view - it allows you to place the interface into any existing page. Your site can be built on ASP.NET, PHP platform or contain only static content. HTTP Commander may be installed on the same server or on another one. With just one line of code, you can place HTTP Commander interface inside a region on any page. See how it looks on our site in the image below or at our website or even at your local PC!

How to use the iframe view mode:
  • Install application and check if it works in stand-alone browser window.
  • Set "IsEmbeddedtoIFRAME" parameter to the "true" at Application settings.

    For this purpose set a parameter value of ShowFullScreenOrExitFullScreenButton in true.

  • You can also place HTTP Commander interface in stand-alone browser window inside DIV tag.
    • Open in notepad on editing page Default.aspx
    • in the Default.aspx file find a line
      <!-- Opening embedded tag DIV -->
      and make sure there is a line below:
      <div id="httpCommander" style="position:relative !important;width:800px;height:600px;left:50px;top:50px;">
      Also find line
      <!-- Closing embedded tag DIV -->
      and make sure there is a line below with closing DIV tag:
      </div>
    • You also can change attributes of style at the DIV element: width, height, etc.
      Note! It is recommended not to change attribute of position style.
    • Save and close.
      Screenshot

    • Change voluntarily in settings DefaultViewMode parameter value:
      InsideDiv - in case of start of HTTP Commander will be shown in an element DIV as it is defined in Default.aspx.
      FullScreen - HTTP Commander will be unrolled on all screen of the browser together with the DIV element.
    • Also for switching between a fullscreen mode and normal to use parameter ShowFullScreenOrExitFullScreenButton.
