HTTP Commander system requirements

Server side requirements
  • HTTP Commander in iframe mode can be placed in any site and server OS: PHP, ASP.NET, clear HTML, Any CMS, Windows, Unix, etc.
  • HTTP Commander application requires Windows OS: Windows Servers 2003, 2003 R2, 2008, 2008 R2, 2012, 2012 R2, 2016; Windows Vista, 7, 8, 8.1, 10; 32 or 64 bit with IIS 6, 7, 7.5, 8, 8.5 or 10 installed
  • .NET Framework 4.0, 4.5 or 4.6
  • Application uses HTTP port (80, 8080, etc.), SSL HTTP Port if SSL used (443) and LDAP port 389 (if LDAP provider to get user membership used) for its work.
Client side requirements
  • Any OS.
  • Internet browser. All popular browsers like Internet Explorer 8+, Edge, Firefox, Chrome, Opera, Safari are supported.
  • JavaScript and CSS enabled.
  • Cookies enabled (for session support).
Mobile devices (smartphones) requirements
  • HTTP Commander supports most popular OS of new and old smartphones/touchscreen PC: iPhone(Apple), iPad, Android-based (Google), BlackBerry (RIM), Windows Mobile, Windows Phone, Symbian, Bada. It is >90% of smartphones market
  • Forms or Basic authentication is supported (include Windows Integrated Authentication)
