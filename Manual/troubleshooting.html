Troubleshooting

The HTTP Commander has some ways to catch errors and check if the application works correctly. Most of the problems are related to the NTFS permissions so check if the permissions are set correctly. Read the FAQ to find answers for some of the known problems.

How to check the application and catch error:
  • Diagnostics: It is a web page that shows the application info. Open "/HTCOMNET/Diagnostics.aspx" page in a browser or click Diagnostics button in the toolbar of the Admin panel interface. To allow running diagnostics for any user, set the parameter ShowDiagnosticsASPXOnlyForAdmins to "false" at Application settings. Otherwise user should be an Administrator, i.e his name should be listed in Administrators key in the Application settings.
    Some information is only shown when special parameters are passed to the page. Query string parameters:
    • showlogons=quantity Number of logged-on unique users.
    • showlogons=names Lists logged-on unique users.
  • Errors handling: The application handles most errors and writes some of them into the Data\Errors.xml file. Check if the application identity has written the permissions into this file.
    If you want to write absolutely all errors you can modify the Global.asax file: Find the //ErrorLog.Add(objErr); line in it and remove "//" symbols in the beginning (i.e. uncomment this line; by default already logged all errors, line uncommented).
